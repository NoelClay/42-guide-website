  Part 2 선행 질문 리스트
  

A. VirtualBox 실제 VM 생성 (마무리 파트)
	- A1. Skip Unattended Installation 을 왜 체크해야 하나? 체크를 안 하면 뭐가 자동 진행되고 왜 불리한가? 42 과제는 직접 세팅을 해보는게 목적이다. 그런데 Unattended Installation은 자동 설치 를 진행하는건데 자동설치를 스킵한다는것은 우리가 일일히 유저 세팅이나, 디스크 파티션 등을 직접 수행한다는것이다. 42 과제할때 유리하다. 안그러면 직접 터미널에 들어가서 세팅을 변경하는 등 여간 귀찮은것이 한두 가지가 아니다.
	- A2. 42 경산/서울 환경에서 home 용량 제한·sgoinfre 등 파일 저장 위치 결정 시 실제 고려해야 할 점은? sgoinfre는 자기한테 부여된 원격 저장소이므로 클러스터 PC를 옮겨 앉아서 해도 유지되게끔 적절하게 사용하라고 있는거지만, 42 경산의 경우에는 알타리안을 이용해서 구매하지 않는 이상 sgoinfre를 이용할 수 없었다. 나는 USB에 vdi를 설치해서 사용하였다. 
B. OS 부팅·사용자·호스트설정
	- B1. hostname 규칙 login42 은 어디(Subject)에 명시되어 있고 왜 이렇게 강제하나? hostname은 사용자42로 고정이다. [text](rank-01/born2beroot/_reference/subject_ko_Born2beRoot.md) 를 보면 가이드가 나와 있다.
	- B2. 도메인 이름을 왜 건너뛰나? 입력하면 시스템이 어떻게 달라지는가? 모르겠음
	- B3. root 비밀번호 vs 사용자 비밀번호 vs (파티셔닝 단계의) LUKS 암호화 비밀번호 — 이 셋의 역할과 사용 시점이 어떻게 다른가? 
		1.root: sudo 권한 따위를 이용할때 필요 관리자 계정 권한 혹은 로그인 접근용
		2. 사용자: 일반 사용자 로그인용
		3. LUKS는 파티셔닝 용, 부팅용. 부팅할때 하드웨어 접근해야되니까.

C. 파티셔닝의 핵심 — 암호화 + LVM 3층 스택 (가장 중요)
	- C1. "Guided - use entire disk and set up encrypted LVM" 에서 "Guided" 는 뭐고 "encrypted LVM" 은 어떤 구조인가?
		encryted 암호화된 + LVM 논리 볼륨 매니저 -> 물리 볼륨 매니저보다 유연함은 유지하면서, 권한 컨트롤이나 용량이 다찼을때 등등 실제 물리 볼륨 매니저처럼 강제성을 부여할수 있다. 논리 볼륨 매니저는 특히 파티셔닝을 중간에 바꿀수도 있기 때문에 매우 선호된다.
		guided는 해당 파티셔닝 프로그램을 돌리는걸 가이드한다고 표현하는거 같다.
	- C2. LVM(Logical Volume Manager) 이 왜 필요한가? 일반 MBR/GPT 파티션과 기능적으로 뭐가 다른가? (PV/VG/LV 용어 포함)
		이미 위에 설명했다. (PV/VG/LV 용어 포함) 저게 뭔 용어인지 전혀 모르겠다. MBR/GPT 파티션 이것도 모른다.
	- C3. LUKS 암호화 는 정확히 어느 계층 에서 작동하나? (LVM 위? 아래? 파일시스템 위?)
		전혀 모르겠다.
	- C4. 멘탈 모델 — 물리 디스크 → LUKS → LVM(PV→VG→LV) → ext4 까지 각 계층이 위·아래 계층에 뭘 제공하고 뭘 받는가?
		전혀 모르겠다.
	- C5. 볼륨 그룹(VG) 크기를 왜 축소해야 추가 LV 공간이 확보되나? 건드리지 않으면 기본값이 어떻게 되어 있기에?
		전혀 모르겠다.
	
D. 파티션/LV 분리 설계 (왜 이 구성인가)
	- D1a. 원 가이드의 "보너스 파트를 위해 /home, /var, /tmp 분리" — 보너스 파트가 정확히 뭘 요구하기에 이 3개여야 하는가?
		/home 용량이 많아야지 유저 서버 용량이 많아지고
		/var 용량이 많아야지 공통 서버 용량이 많아지고
		/tmp 용량이 많아야지 실시간 프로세스량이 늘어나는 만큼 log량도 늘기 때문에 tmp도 확보해야된다.
	- D1b. 일반 서버 관리 관점에서도 이 3개를 분리하는 이유는? (쿼터? 마운트 옵션 차이? 보안·가용성?)
		잘 모르겠다.
	- D1c. 왜 /usr, /boot, /opt 같은 다른 디렉토리는 분리하지 않나?
		모르겠다.
	- D2a. 추가로 만드는 논리 볼륨 /srv — FHS 기준 뭐 하는 디렉토리인가? 본 프로젝트(LEMP + WordPress)에서 실제 쓰나?
		모르겠다.
	- D2b. /var/log 를 별도 LV로 뽑는 이점? (로그 폭주로 / 가 꽉 차는 사고 방지? 모니터링 스크립트와 연관?)
		모르겠다.
	- D2c. LV로 분리하는 것과 그냥 디렉토리로 두는 것의 실질적 차이는?
		모르겠다.
	- D3a. Journaling file system 이 정확히 뭘 의미하나? (쓰기 전에 저널에 기록 → 크래시 복구)
		모르겠다.
	- D3b. ext4 vs xfs vs btrfs — 왜 Born2beRoot에서 ext4 인가?
		 모르겠다.
	- D3c. 모든 파티션(/, /home, /var, /tmp, /srv, /var/log)에 전부 ext4인가? /boot 는 어떻게 되어 있나?
		모르겠다.
	
E. 설치 마무리 단계

	- E1. Debian 아카이브 미러 란 정확히 뭐고, 프록시 설정은 언제·왜 건드리나? (학교 네트워크·사내망?)
		아카이브 = 보관소, 미러 = 다운받게 해주는 서버, 프록시 설정은 이제 vm과 같은 네트워크 상에 있는 다른 클라이언트가 접근하게 하려면 포트도 충돌안나게 바꿔주는 등의 작업해줘야되기 때문에 필요하다. 포트 포워딩이랑 똑같다.
	- E2. popularity-contest(패키지 사용 설문) 이 뭐고 왜 거부하나? 거부해도 되는 근거는?
		잘 모르겠다.
	- E3. GRUB 로더 — 뭐고 왜 필수인가? 암호화된 LVM과의 관계는 어떻게 되나? (GRUB 단계에서 LUKS 암호를 푸는 것인지, /boot 파티션이 별도로 암호화 바깥에 있는지 등)
		GRUB 로더가 펌웨어처럼 설치되어 있으면 실행 옵션 런쳐 역할을 해준다. (GRUB 단계에서 LUKS 암호를 푸는 것인지, /boot 파티션이 별도로 암호화 바깥에 있는지 등) 이건 잘 모르겠다.
	- E4. 소프트웨어 선택에서 SSH server + standard system utilities 만 체크 — 왜 이 둘만? standard system utilities 에는 정확히 뭐가 포함되나? 다른 옵션(예: web server, print server)을 체크하면 안 되는 이유?
		그래픽 데스크탑 관련 패키지를 설치 안하려 하기 때문이다.
