mysql: innodb(128MiB)(데이터 캐싱 기본 용량)(출처: 전역 버퍼(Global Buffers)의 고정 기본값

  * 출처: MySQL 8.0 Reference Manual - 15.8.3.1 Configurable InnoDB Read-Only Variables
    ([https://dev.mysql.com/doc/refman/8.0/en/innodb-parameters.html\#sysvar\_innodb\_buffer\_pool\_size](https://dev.mysql.com/doc/refman/8.0/en/innodb-parameters.html#sysvar_innodb_buffer_pool_size))
  * 직시 데이터: innodb\_buffer\_pool\_size
      * Default Value: 134217728 bytes (128 MiB) + max\_connections(151개)([maxconnection](https://dev.mysql.com/doc/refman/8.4/en/server-system-variables.html#sysvar_max_connections)) \* 2MB
  * 직시 데이터 (파츠):
      * max\_connections: Default 151
      * thread\_stack: Default 286720 bytes (약 256 KB) (64-bit 시스템 기준)
      * net\_buffer\_length: Default 16384 bytes (16 KB) \* complexqueries(2MB)(출처: 복잡한 쿼리(정렬, 읽기 등) 수행 시 개별 세션 버퍼가 동적 할당되어 연결당 피크 시 약 1\~2MB의 메모리를 점유함. 출처: [이미지](https://www.percona.com/blog/understanding-mysql-memory-usage-with-performance-schema/) = 430MB -\> 여유를 위해 \*2 = 860MB
      * 최대값으로 그냥 더하는게 안되는 이유는 시스템이 실제로 그렇게 메모리 용량을 차지하고 있지 않을뿐더러 빠르게 해소되고나서 0mb에 가깝게 운용되기 때문[stackoverflow](https://stackoverflow.com/questions/65780309/mysql-per-thread-memory-variables-that-lead-to-thread-memory-usage/65781074#65781074)
